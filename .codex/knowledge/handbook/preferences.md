# User Preferences

Owned by Master-1 (Interface). Updated when the user expresses preferences.

## Communication Style

- User prefers direct, concise communication — no filler, lead with the answer
- User values rundowns and structured summaries over narrative explanations

## Domain Priorities

- Knowledge system quality and iterative improvement are high priority
- Credit burn reduction and operational efficiency are high priority

## Approval Preferences

- User wants to be consulted before major architectural changes
- User prefers planning mode before execution for multi-file changes
- Research tasks should be sent to deep research agent, not executed inline

## Design Philosophy

- Intelligence does the thinking, economy does the typing
- Agents should accumulate insight over iterations — knowledge is a first-class product
- Cross-research continuity: never re-research something already researched in the same project
- Condensation should happen by the agent with context, not a separate summarizer
